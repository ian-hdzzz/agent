  <name>Task 1: Create workflows router with CRUD endpoints</name>
  <files>backend/app/api/workflows.py</files>
  <action>
Create `backend/app/api/workflows.py` following the pattern in `agents.py`:

1. **Pydantic Models:**
   - `WorkflowResponse`: id, name, description, config (parsed JSON), is_enabled, created_at, updated_at, created_by
   - `WorkflowCreateRequest`: name, description (optional), config (dict)
   - `WorkflowUpdateRequest`: name (optional), description (optional), config (optional dict)
   - `WorkflowListResponse`: List[WorkflowResponse]

2. **Endpoints:**
   - `GET /` - List workflows for current user (filter by created_by = user.user_id)
   - `GET /{workflow_id}` - Get single workflow by UUID
   - `POST /` - Create workflow (requires OperatorUser)
   - `PUT /{workflow_id}` - Update workflow (requires OperatorUser, must be owner)
   - `DELETE /{workflow_id}` - Delete workflow (requires OperatorUser, must be owner)

3. **Implementation details:**
   - Use `Flow` model from `app.db.models`
   - Store config as JSON string in `config_yaml` column (use `json.dumps/loads`)
   - Use `DbSession`, `CurrentUser`, `OperatorUser` from deps
   - Return 404 if workflow not found
   - Return 403 if user tries to modify workflow they don't own
   - Router prefix: `/workflows`, tags: `["Workflows"]`
  </action>
