   - Import field components from ./fields/
   - Props: defaultValues, onChange (called on every valid change, debounced)

   - Use useEffect + debounce (300ms) to call onChange when form changes
   - Render:

     - TemperatureSlider
     - MaxTokensInput
     - "Advanced Parameters" section heading (visible, NOT collapsed):
       - TopPSlider
       - TopKInput
       - StopSequencesInput

2. Create `properties-panel/index.tsx`:
   - Fixed right sidebar: className="w-96 border-l border-secondary bg-primary h-full overflow-y-auto"
   - Header with node label and close button (X icon)
   - Import useBuilderStore from '../store'
   - Get selectedNodeId and updateNode from store: `const { selectedNodeId, nodes, updateNode } = useBuilderStore()`
   - Find selectedNode: `const selectedNode = nodes.find(n => n.id === selectedNodeId)`
   - Extract defaultValues from selectedNode.data.component for AgentConfigForm
   - Implement onChange handler that:
     a. Receives AgentFormValues from form
     b. Maps to Component<AgentConfig> format
     c. Calls updateNode(selectedNodeId, { component: updatedComponent })
   - Conditional render: if agent node, show AgentConfigForm with defaultValues and onChange
   - For non-agent nodes, show existing placeholder fields (label, description)
   - Props: onClose (for X button)

3. Update `frontend/components/builder/builder.tsx`:
   - REMOVE the Drawer component entirely (search for Drawer import and usage around selectedNodeId)
   - Remove Drawer-related imports if no longer used
   - Add PropertiesPanel as fixed right element inside the Layout
   - Position: after the Content section, conditional on selectedNodeId
   - Pass onClose={() => setSelectedNode(null)} to PropertiesPanel
   - Layout structure: flex row with canvas content + optional PropertiesPanel

4. Update `frontend/types/datamodel.ts`:
   - Extend AgentConfig to include all configurable fields:
     - topP?: number
     - topK?: number
     - stopSequences?: string[]
   - Ensure ClaudeModel type includes new model IDs (or make model a string for flexibility)

5. Store wiring (CRITICAL - this is how persistence works):
   - PropertiesPanel.index.tsx reads selectedNode from useBuilderStore
   - When form values change -> onChange -> updateNode(nodeId, newData)
   - updateNode modifies nodes array in Zustand store
   - When user clicks different node -> selectedNodeId changes -> PropertiesPanel re-reads from store -> form shows persisted values
   - This ensures settings persist through deselect/reselect cycle
