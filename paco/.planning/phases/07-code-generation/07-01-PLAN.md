  <behavior>
    The workflowToIR function transforms React Flow canvas state into a WorkflowIR object.

    Input:
    - nodes: CustomNode[] (from React Flow/Zustand store)
    - edges: CustomEdge[] (connections between nodes)
    - workflowName: string
    - workflowDescription?: string

    Output: WorkflowIR object containing:
    - name, description
    - agent config (model, systemPrompt, temperature, maxTokens, topP, topK, stopSequences)
    - mcpServers[] (connected MCP server nodes with enabledTools)
    - allowedTools[] (built-in tools from agent config)
    - knowledgeSources[] (connected knowledge nodes)
    - permissionMode

    Test Cases:
    1. Empty workflow (only team node) -> throws "Workflow must have exactly one agent node"
    2. Single agent node -> returns IR with agent config, empty mcpServers/knowledgeSources
    3. Agent + MCP server connected via edge -> mcpServers populated with server config
    4. Agent + multiple MCP servers -> all connected servers in mcpServers array
    5. Agent + knowledge node connected -> knowledgeSources populated
    6. Full workflow (agent + MCP + knowledge) -> complete IR
    7. Agent with all optional fields (topP, topK, stopSequences) -> all fields in IR
    8. MCP server with enabledTools filter -> enabledTools array in IR
  </behavior>
  <implementation>
    1. Create ir.ts with WorkflowIR, AgentIR, McpServerIR, KnowledgeSourceIR interfaces
    2. Create workflow-to-ir.ts with workflowToIR function:
       - Find agent node using isAgentComponent guard
       - Find connected MCP nodes via edges with connectionType 'mcp'
       - Find connected knowledge nodes via edges with connectionType 'knowledge'
       - Map node configs to IR format
       - Return complete WorkflowIR
    3. Create index.ts barrel export
    4. Write tests covering all cases
  </implementation>
</feature>
