  <behavior>
    The generateTypeScriptCode function transforms WorkflowIR into valid TypeScript code.

    Input: WorkflowIR object

    Output: string containing valid TypeScript code that:
    - Imports from @anthropic-ai/claude-agent-sdk
    - Uses async main() with for await...of query() pattern
    - Includes systemPrompt if specified
    - Includes model if not default
    - Includes mcpServers config for connected servers
    - Includes allowedTools array (built-in + mcp tool wildcards)
    - Includes permissionMode
    - Properly escapes backticks and $ in user strings
    - Adds header comment with workflow name/description
    - Renders knowledgeSources as documentation comments (Claude Agent SDK does not have built-in RAG, so knowledge sources are listed as reference documentation for manual integration)

    Test Cases:
    1. Minimal IR (agent only) -> valid TS with just model/permissionMode
    2. IR with systemPrompt -> systemPrompt in options
    3. IR with stdio MCP server -> mcpServers with command/args
    4. IR with SSE MCP server -> mcpServers with type/url
    5. IR with multiple MCP servers -> all servers in config
    6. IR with enabledTools -> mcp__servername__toolname in allowedTools
    7. IR with enabledTools=[] (wildcard) -> mcp__servername__* in allowedTools
    8. SystemPrompt with backticks and $ -> properly escaped
    9. Full IR (all fields) -> complete valid TypeScript
    10. IR with knowledgeSources -> comment block listing knowledge sources
  </behavior>
  <implementation>
    1. Create ir-to-typescript.ts with:
       - generateTypeScriptCode(ir: WorkflowIR): string
       - Helper: generateMcpServersConfig(servers)
       - Helper: generateAllowedTools(ir)
       - Helper: escapeTemplateString(str)
       - Helper: generateKnowledgeSourcesComment(sources) - renders sources as JSDoc comment
    2. Use template literals for code generation
    3. Handle optional fields with conditional inclusion
    4. If knowledgeSources present, add comment block at top:
       ```typescript
       /**
        * Knowledge Sources (for manual RAG integration):
        * - Document: internal-docs (files: doc1.pdf, doc2.md)
        * - URL: api-reference (https://api.example.com/docs)
        *
        * Note: Claude Agent SDK does not include built-in RAG.
        * Integrate these sources using your preferred retrieval system.
        */
       ```
    5. Update index.ts to export generateTypeScriptCode
    6. Write tests covering all cases including knowledge sources
  </implementation>
</feature>
